Phase 5 implementation in src/cmux-notify.ts:

  <how-to-verify>
**Automated checks (run first — confirm these pass before testing manually):**
```bash
npm run typecheck && npm test && npm run build
```

**Manual smoke tests (run inside a live cmux session):**

**Test 1 — Session working status:**
Start an OpenCode session and give it a task. The sidebar should show an amber "Working" status. When the session finishes, the status should clear and a "Done" desktop notification should appear.

**Test 2 — Permission gate (if you can trigger one):**
Give OpenCode a task that requires permission (e.g., run a shell command). The sidebar should switch to red "Waiting" when the permission request appears. A desktop notification should arrive titled "OpenCode" with subtitle "Needs Permission". After approving, the sidebar should restore to "Working" (if session is still busy) or clear (if idle).

**Test 3 — Sidebar disabled:**
Set `"sidebar": { "enabled": false }` in `~/.config/opencode/opencode-cmux.json`. Restart OpenCode. Run a session — no `cmux set-status` or `cmux clear-status` calls should fire. Done notify should still work (it's gated separately).

**Test 4 — Permission notify disabled:**
Set `"notify": { "permissionRequest": false }`. Trigger a permission request. The sidebar should still update (waiting → red), but no desktop notification should appear.

**Test 5 — isWaitingForInput gate:**
If a session goes idle while a permission or question is still pending (e.g., multi-step workflow where agent finished but permission modal is still open), no "Done" notify should fire and the sidebar should NOT clear.

**What to look for:**
- Sidebar key is "opencode" (not "agent-N")
- Subagent sessions (CmuxSubagentViewer panes) should NOT show sidebar status from cmux-notify.ts handlers (they're filtered by isSubagent())
- Done notify fires exactly once per idle transition, not on busy→busy cycles
