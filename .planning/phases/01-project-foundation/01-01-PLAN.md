  <action>
Create `tsconfig.json` at project root with these EXACT settings:

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "lib": ["ESNext"],
    "types": ["bun-types"],
    "outDir": "./dist",
    "declarationDir": "./dist",
    "declaration": true,
    "declarationMap": true,
    "strict": true,
    "skipLibCheck": true,
    "esModuleInterop": false,
    "isolatedModules": true,
    "verbatimModuleSyntax": true
  },
  "include": ["src/**/*.ts"],
  "exclude": ["node_modules", "dist"]
}
```

Key rationale:
- `"moduleResolution": "bundler"` — required for Bun; enables bare imports without file extensions in type-checking
- `"types": ["bun-types"]` — provides `$` shell tag type (used in cmux-notify.ts and plugin-base.ts)
- `"verbatimModuleSyntax": true` — enforces `import type` for type-only imports (already used in source)
- `"declarationMap": true` — provides source map from .d.ts back to .ts for editor navigation
- `"include": ["src/**/*.ts"]` — source is in src/ (not lib/ — the codebase map is outdated)
  </action>

  </verify>
  <done>tsconfig.json exists with outDir=./dist, target=ESNext, moduleResolution=bundler, types=bun-types, strict=true</done>
</task>
